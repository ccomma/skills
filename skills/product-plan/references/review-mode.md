# Review Mode

Use this when the user wants a diagnosis of document responsibilities, ownership drift, or structural quality before authorizing full repair.

## Goal

Review mode should identify:

- where responsibilities are blurred
- which layers are missing
- which layers are already strong
- which fixes are minimum, medium, or large
- which issues point back to the project versus the skill design

## Output Shape

Prefer a concise report with:

- project size judgment
- existing quality judgment
- strongest layers to preserve
- main ownership or loading problems
- missing governance or template layers
- recommended repair order

## Stop Rule

Unless the user requested direct repair, stop after diagnosis and recommendations. Do not automatically create or rewrite the whole stack.
